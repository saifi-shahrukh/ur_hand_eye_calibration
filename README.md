# UR Hand-Eye Calibration (Eye-in-Hand)

Sub-millimetre hand-eye calibration for **Universal Robots** with an
**Intel RealSense** camera mounted on the gripper (eye-in-hand).

Tested with UR5e + Robotiq Hand-E + RealSense D435.
Works with any UR robot that exposes the RTDE interface.

---

## Transform Convention

Convention: T_dest_src → p_dest = T_dest_src @ p_src

Frame chain (calibration solves for T_tcp_cam):

Board ──T_cam_board──▸ Camera ──T_tcp_cam──▸ TCP ──T_base_tcp──▸ Base ▲ ▲ solvePnP calibrateHandEye (per frame) (output)

Camera point → Base: p_base = T_base_tcp @ T_tcp_cam @ p_cam ────────── ────────── from RTDE calibration

| Symbol | Direction | Source |
|---|---|---|
| `T_base_tcp` | TCP → Base | `getActualTCPPose()` via RTDE |
| `T_tcp_cam` | Camera → TCP | `cv2.calibrateHandEye()` output |
| `T_cam_board` | Board → Camera | `cv2.solvePnP()` per frame |

---

## Hardware Requirements

- Universal Robot (UR3/5/10/16/20/30e) with RTDE enabled
- Intel RealSense D400 series (D415/D435/D455) mounted on gripper
- Printed ChArUco calibration board (generated by Step 0)
- (Optional) Robotiq gripper for pick-and-place demo

## Software Requirements

- Python 3.8+
- OpenCV 4.7+ (with `cv2.aruco` module)
- See `requirements.txt` for full list

Step-by-Step Guide
Step 0 — Generate ChArUco Board
python scripts/00_generate_board.py
Saves a printable PNG in results/charuco_board.png. Print at 100 % scale and mount on a flat, rigid surface. Measure the printed square size with calipers and update config.yaml.
Step 1 — Camera Intrinsics

### Option A: factory intrinsics from RealSense (fast, less accurate)

python scripts/01_calibrate_intrinsics.py --factory

### Option B: full calibration with ChArUco board (recommended)

python scripts/01_calibrate_intrinsics.py

Hold the board in front of the camera at varied distances and angles. Press c to capture, q when done (≥ 15 images recommended).
Step 2 — Collect Samples
python scripts/02_collect_samples.py

    Interactive gripper mask setup — slide bars to hide gripper fingers.
    Move the robot so the board is visible from different poses.
    Press c to capture. Quality checks run automatically.
    Collect 25–40 diverse samples.

Step 3 — Compute Calibration

Runs automatically after Step 2, or re-run from saved samples:

python scripts/03_compute_calibration.py
python scripts/03_compute_calibration.py --samples results/samples/samples_latest.json

Step 3b — Optimise (Optional)

Try every combination of sample filters to find the best subset:

python scripts/03b_optimize_calibration.py --target-pos-mm 2.0 --target-rot-deg 0.5

Step 4 — Verify
python scripts/04_verify_calibration.py
Move the robot to ≥ 5 different poses with the board visible. Press r at each pose to record. The board position in base frame should stay constant.
Good: std < 3 mm | Excellent: std < 1 mm
Step 5 — Live Test
python scripts/05_live_test.py

Click on the image to transform a pixel to base-frame coordinates.
Configuration (config/config.yaml)
Key	Description
robot.ip	Robot IP address
camera.width/height/fps	RealSense resolution
gripper_mask.*_pct	Percentage of image to mask (gripper fingers)
charuco.*	Board dimensions — must match printed board
calibration.max_reproj_error_px	Reject detections above this
calibration.target_position_error_mm	Target accuracy
Troubleshooting
Problem	Solution
QFontDatabase warnings	Harmless — suppressed by env vars in scripts
High reprojection error	Increase gripper mask, improve lighting
numpy.bool_ JSON error	Update src/calibrator.py (fixed in this repo)
Position error > 5 mm	More diverse poses, check board flatness
Board not detected	Check mask isn't hiding board, adjust thresholds
Robot connection refused	Enable RTDE in PolyScope, check firewall
